# Contribution Guidelines

Thank you for helping make this list better!

## Adding a resource

- **Free/open first.** Commercial tools are accepted only if they have a genuinely usable free tier, and must be marked as such.
- **One line of "why".** Every entry needs a short description explaining why it is useful to an industrial engineer or operations professional.
- **Working links only.** Prefer canonical sources (official repo or site). No redirects, no archived or unmaintained projects.
- **Format:** `- [Name](link) - Description ending with a period.` (plain hyphen separator).
- **Fits an existing section.** If not, propose a new section in your PR description and place it where it fits best.
- No affiliate links, no self-promotion without substance.

## What we don't accept

- Paywalled content without a meaningful free tier
- Unmaintained or archived projects
- Listicle blog posts and SEO content farms
- Bulk or template-generated pull requests — write your PR by hand, one change per PR

## How

1. Fork the repo
2. Add your entry following the format above
3. Open a pull request with a one-line explanation of why the resource earns its place

Small fixes (typos, dead links) are always welcome as direct PRs.
